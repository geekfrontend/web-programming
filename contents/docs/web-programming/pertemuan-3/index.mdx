## 1) 🚀 Instalasi dan Setup React + Vite

### Prasyarat

- Node.js versi LTS sudah terpasang.
- npm sudah tersedia dari instalasi Node.js.
- Terminal bisa menjalankan `node -v` dan `npm -v`.
- Editor kode siap digunakan.

### Langkah Setup Detail

```bash
node -v
npm -v
npm create vite@latest note-app -- --template react-ts
cd note-app
npm install
npm run dev
```

### Verifikasi Hasil Instalasi

1. Setelah `npm run dev`, terminal menampilkan URL lokal.
2. Buka URL tersebut di browser.
3. Jika halaman React + Vite tampil, setup berhasil.
4. Ubah teks di `src/App.tsx`, simpan, lalu cek browser untuk memastikan hot reload berjalan.

### Perintah Tambahan

```bash
npm run build
npm run preview
```

- `npm run build` menghasilkan build production.
- `npm run preview` menjalankan hasil build secara lokal.

### Fungsi folder utama

- `src`: tempat utama kode aplikasi React, termasuk entry app dan styles.
- `public`: file statis yang bisa diakses langsung tanpa import dari `src`.
- `node_modules`: kumpulan package dependency yang diinstall npm.
- `dist`: hasil akhir build production, folder ini muncul setelah `npm run build`.

### Penjelasan file konfigurasi

- `vite.config.ts`: konfigurasi Vite seperti plugin, alias, dan opsi server/build.
- `package.json`: daftar script (`dev`, `build`, `preview`) dan dependency.
- `eslint.config.js`: aturan linting agar kualitas kode konsisten.
- `index.html`: entry point utama yang memuat aplikasi React.
- `tsconfig.json`: konfigurasi TypeScript utama untuk referensi project.
- `tsconfig.app.json`: konfigurasi TypeScript khusus kode aplikasi frontend.
- `tsconfig.node.json`: konfigurasi TypeScript untuk file tooling Node seperti Vite config.

### Best practices organisasi proyek

- Buat folder `src/components` saat mulai memecah UI jadi komponen kecil.
- Buat folder `src/types` untuk tipe data yang dipakai lintas komponen.
- Simpan asset lokal di `src/assets` atau `public` sesuai kebutuhan.
- Simpan helper function di `src/utils` saat logic mulai bertambah.
- Gunakan penamaan file konsisten: `PascalCase` untuk komponen.
- Pisahkan style global (`index.css`) dan style komponen (`App.css`) agar maintainable.

## 3) 🧩 Tutorial Bertahap: Build Note App TypeScript

Urutan di bawah mengikuti alur kerja pengembangan: buat file → isi kode → integrasi → verifikasi.

### Langkah 1 — Siapkan tipe data utama

**Tujuan**

- Menentukan kontrak data catatan agar konsisten.

**Buat file baru** `src/types/note.ts`

```ts
export type Note = {
  id: string;
  title: string;
  body: string;
};
```

**Verifikasi**

- File tersimpan tanpa error.
- Tidak ada tanda merah pada editor TypeScript.

### Langkah 2 — Buat komponen header

**Tujuan**

- Menampilkan identitas halaman Note App.

**Buat file baru** `src/components/NoteHeader.tsx`

```tsx
export function NoteHeader() {
  return (
    <header className="page-header">
      <h1>Note App</h1>
      <p>Catat ide, tugas, dan ringkasan kelas dalam satu tempat.</p>
    </header>
  );
}
```

**Integrasi**

- Import `NoteHeader` ke `src/App.tsx`.
- Render `<NoteHeader />` di dalam `<main>`.

**Verifikasi**

- Jalankan `npm run dev`.
- Pastikan judul dan deskripsi muncul di browser.

### Langkah 3 — Buat komponen kartu catatan

**Tujuan**

- Menampilkan satu item catatan dengan props bertipe.

**Buat file baru** `src/components/NoteCard.tsx`

```tsx
type NoteCardProps = {
  title: string;
  body?: string;
};

export function NoteCard({
  title,
  body = "Belum ada isi catatan.",
}: NoteCardProps) {
  return (
    <article className="note-card">
      <h3>{title}</h3>
      <p>{body}</p>
    </article>
  );
}
```

**Integrasi**

- Coba render satu `NoteCard` langsung di `App.tsx` untuk uji cepat.

**Verifikasi**

- Card tampil dengan benar.
- Saat `body` tidak dikirim, teks default tampil.

### Langkah 4 — Buat komponen daftar catatan

**Tujuan**

- Merender banyak catatan menggunakan `map`.

**Buat file baru** `src/components/NoteList.tsx`

```tsx
import { NoteCard } from "./NoteCard";
import type { Note } from "../types/note";

type NoteListProps = {
  notes: Note[];
};

export function NoteList({ notes }: NoteListProps) {
  return (
    <section className="note-list">
      {notes.map((note) => (
        <NoteCard key={note.id} title={note.title} body={note.body} />
      ))}
    </section>
  );
}
```

**Integrasi**

- `NoteList` akan dipanggil oleh `App.tsx` dengan props `notes`.

**Verifikasi**

- Tidak ada warning `key` di console browser.
- Semua item array ditampilkan sebagai card.

### Langkah 5 — Satukan semua komponen di App

**Tujuan**

- Menghubungkan data + header + list dalam satu halaman.

**Ubah file** `src/App.tsx`

```tsx
import { NoteHeader } from "./components/NoteHeader";
import { NoteList } from "./components/NoteList";
import type { Note } from "./types/note";

const notes: Note[] = [
  { id: "n-1", title: "Setup proyek", body: "Gunakan Vite + React." },
  { id: "n-2", title: "Desain UI", body: "Buat layout dengan CSS." },
  { id: "n-3", title: "Rencana fitur", body: "Tambah CRUD note." },
];

export default function App() {
  return (
    <main className="page">
      <NoteHeader />
      <NoteList notes={notes} />
    </main>
  );
}
```

**Penjelasan komponen utama**

- `NoteHeader`: menampilkan judul dan deskripsi aplikasi.
- `NoteCard`: template card untuk satu catatan.
- `NoteList`: mengubah array data menjadi daftar card.
- `App`: orchestrator yang menggabungkan semua komponen.

**Verifikasi**

- UI menampilkan header dan 3 card.
- Ubah isi salah satu object di `notes`, lalu simpan.
- Pastikan perubahan langsung terlihat di browser.

### Langkah 6 — Verifikasi setelah setiap pembuatan file

Pakai checklist ini agar kualitas tetap aman:

1. Simpan file dan cek error di editor.
2. Pastikan dev server tidak menampilkan compile error.
3. Cek tampilan di browser sesuai tujuan langkah.
4. Coba ubah data kecil untuk memastikan integrasi aktif.
5. Jika error muncul, cek import path dan tipe props dulu.

## 4) 🧠 Ringkasan Fundamental yang Dipakai

### TSX

- TSX adalah JSX dengan TypeScript.
- Ekspresi JavaScript tetap memakai `{}`.
- Keunggulan utama: deteksi error tipe lebih awal.

### Component

- Functional component adalah pola utama.
- Setiap komponen fokus pada satu tanggung jawab.
- Nama komponen gunakan PascalCase.

### Props

- Props mengalir dari parent ke child.
- Type props membuat kontrak data jelas.
- Optional props + default value mencegah UI kosong.

### Prop Drilling

- Terjadi saat props harus lewat banyak level.
- Jika struktur makin dalam, gunakan Context API.

## 🛠️ Troubleshooting Umum

### 1. `npm create vite@latest` gagal dijalankan

- Cek koneksi internet.
- Cek versi Node.js.
- Coba ulang perintah setelah update npm:

```bash
npm install -g npm
```

### 2. Port development sudah dipakai

- Hentikan proses yang memakai port tersebut.
- Atau jalankan ulang dev server agar Vite memilih port lain.

### 3. Modul tidak ditemukan

- Pastikan `npm install` sudah selesai.
- Cek penulisan path import.
- Cek huruf besar-kecil nama file.

### 4. Perubahan tidak muncul di browser

- Simpan file dan lihat log terminal.
- Hard refresh browser.
- Restart `npm run dev` jika perlu.

### 5. Build production gagal

- Jalankan lint atau pengecekan error terlebih dahulu.
- Periksa error pada import, sintaks TSX/JSX, atau dependency yang belum terpasang.
